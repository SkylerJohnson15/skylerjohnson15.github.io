Case Study: How AtlasDocs Works

Problem

Teams store important information in PDFs and internal docs, but searching through them is slow and inconsistent. The goal was to enable fast Q&A with answers grounded in the original document text.

Approach

I implemented a RAG pipeline: chunk documents, generate embeddings, retrieve the most relevant chunks via vector search, then use an LLM to generate a clear answer with citations.

Architecture

Upload PDF/TXT → Chunk → Embed → Vector Search → LLM Answer + Citations

What I Learned

  • How chunk size/overlap affects retrieval quality
  • How to keep answers grounded using citations + context limits
  • How to deploy an AI app and manage secrets safely
Outcome: A deployable AI assistant demo that supports multi-document Q&A with citations, mobile UX improvements, and a clean product-style interface.
